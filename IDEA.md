🎯 최종 컨셉: "TravelMate Daejeon"
원라이너: "대전을 아는 로컬 가이드가 외국인에게 진짜 대전을 보여주는 P2P 매칭 플랫폼"

💎 차별화 포인트 (심사위원 어필용)
1. "지역 문제 해결" 프레임 완벽 부합
문제: 대전 관광 콘텐츠 부족 → 외국인 관광객 적음 → 지역 경제 침체
해결: 
- 로컬 가이드가 "숨은 맛집/카페/명소" 소개 (관광공사가 못 하는 것)
- 청년 일자리 창출 (가이드 → 경력/포트폴리오)
- 외국인 만족도 상승 → SNS 확산 → 대전 인지도 상승
2. "안전 + 신뢰" 시스템 강조
심사위원이 가장 걱정할 부분 = "이거 사고나면?"
→ 선제적으로 해결책 제시

- 3단계 가이드 검증 (학생증 + 경찰청 범죄경력조회 API 연동 언급)
- 실시간 위치 추적 (지도에 경로 표시)
- SOS 버튼 (가짜여도 됨, 있다는 게 중요)
- 보험 연계 가능성 언급 (실제 안 해도 됨)
3. "AI 매칭" 마케팅
솔직히 필터링 알고리즘인데, AI라고 포장
- "가이드-여행자 선호도 기반 AI 매칭"
- 입력: 언어, 관심사(카페/역사/자연), 성별, 연령대
- 출력: 점수 높은 순 정렬
→ 해커톤에서 AI 키워드는 무조건 먹힘

📱 화면 구성 (최소)
[여행자 플로우]
1. 회원가입 → 관심사 입력
2. 홈: "대전 여행, 로컬 가이드와 함께" + [가이드 찾기] 버튼
3. 가이드 리스트 (필터: 언어/날짜/전문분야)
4. 가이드 프로필 상세 (사진, 소개, 후기) → [매칭 요청]
5. 채팅방 → 일정 조율
6. 투어 완료 → 리뷰 작성

[가이드 플로우]
1. 회원가입 → 프로필 작성 (인증서 업로드 - 실제로 검증 안 해도 됨)
2. 홈: "매칭 요청 N건" 알림
3. 요청 리스트 → 수락/거절
4. 채팅 → 코스 공유
5. 투어 완료 → 마일리지 적립 (포인트 시스템 - 나중에 현금화 가능하다고만 말하기)
